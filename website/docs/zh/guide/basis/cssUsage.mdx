# 引入样式资源

EMP 内置多种常用的样式资源处理方式，包括 Sass 预处理器、CSS Modules、CSS 内联和 CSS 压缩。PostCSS 作为显式扩展插件接入，适合需要复用 PostCSS 生态插件的项目。

除此之外，EMP 也提供了多个配置项来自定义样式资源的处理规则。

## 使用 Sass

EMP目前内置了社区流行的 CSS 预处理器之一：`Sass`。默认情况下，你不需要对 Sass 进行任何配置。目前您无法对其进行任何配置。

## 使用 PostCSS 扩展

EMP v4 默认样式链路不强制依赖 PostCSS。只有当项目需要 `autoprefixer`、`pxtorem`、`pxtovw` 或其他 PostCSS 插件时，才需要安装并注册 `@empjs/plugin-postcss`。

如果只是需要浏览器前缀、现代 CSS 转换或 CSS 压缩，优先评估 `@empjs/plugin-lightningcss`。它基于 `build.browserslist` 处理目标环境，通常可以替代传统 `autoprefixer` 链路。

需要显式接入 PostCSS 时，可以在插件中维护 `postcssOptions.plugins`：

```js title="emp.config.js"
import pluginPostcss, {postcss as postcssPlugins} from '@empjs/plugin-postcss'

export default defineConfig(store => {
  return {
    plugins: [
      pluginPostcss({
        postcssOptions: {
          plugins: [
            postcssPlugins.autoprefixer(),
            postcssPlugins.pxtovw({}),
          ],
        },
      }),
    ],
  }
})
```

### PostCSS 支持边界

- Tailwind v4 默认使用 `@empjs/plugin-tailwindcss`，不需要通过 PostCSS 注册 `tailwindcss`。
- Tailwind 2/3 项目若暂不迁移到 v4，可通过 `@empjs/plugin-postcss` 显式保留 PostCSS 链路。
- 任意 PostCSS 生态插件都需要通过 `@empjs/plugin-postcss` 显式接入。

## CSS 压缩

在生产环境构建时， EMP 会将 CSS、JS 等`静态资源`进行压缩，以达到更好的传输效率。

EMP 默认使用 Rspack 内置的 `SwcCssMinimizerRspackPlugin` 插件，在生产环境构建时自动压缩 CSS 代码。

## 引用 node_modules 里的样式

你可以直接引用 node_modules 里的样式文件。

- 在组件中引入:
  ```tsx title="src/appendFile.tsx"
  // 引用 Arco Design 样式：
  import '@arco-design/web-react/dist/css/arco.css';
  ```
- 在样式文件中引用：
  ```css title="src/App.css"
  /* 引用 normalize.css */
  /* https://github.com/necolas/normalize.css */
  @import 'normalize.css';

  body {
    /* */
  }
  ```

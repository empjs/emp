# EMP 升级说明

## 升级到 EMP v4 Beta

EMP v4 Beta 已切到 Rspack 2 与官方 Module Federation 2.x 底座，核心 `@empjs/*`
包进入统一 v4 beta 版本线。CDN 包和 legacy runtime 包仍保持独立版本线，不随核心包统一升到 v4。

### 安装 v4 beta

<PackageManagerTabs command="add @empjs/cli@beta -D" />

如果项目使用 EMP 官方插件，请优先同步升级到 beta 版本：

```bash
pnpm add @empjs/cli@beta @empjs/plugin-react@beta @empjs/share@beta -D
```

Vue、Tailwind、PostCSS、Lightning CSS、Stylus 等插件按项目实际使用范围升级即可。

### v4 Beta 支持面变化

- 构建底座切到 Rspack 2，EMP 配置仍保留 `defineConfig(...)` 和插件化组织方式。
- Module Federation 默认走 `pluginRspackEmpShare(...)`，业务侧不需要重写已有联邦配置形态。
- Tailwind CSS 默认样例保留 v4，Tailwind v2/v3 不再作为默认示例项目维护。
- 示例应用已收敛为最小真实验收面：Rspack 2、Module Federation、Vue 2/3、React 19、Tailwind v4、adapter canary 和综合 demo。
- `apps/**`、`website`、`@empjs/cdn-*`、`@empjs/lib-*` 不进入核心 beta 统一发布范围。

### 示例与真实验收

v4 Beta 不再维护大量历史 demo。仓库默认保留 13 个 `apps/**` 示例项目，其中 8 个进入默认真实验收：

| 能力 | 默认验收示例 |
| --- | --- |
| Rspack 2 基线 | `rspack2-modern-module` |
| Rspack 2 优化 | `rspack2-optimization` |
| Module Federation 浏览器链路 | `mf-host`、`mf-app` |
| Vue 3 项目验收 | `vue-3-base`、`vue-3-project` |
| Tailwind CSS v4 | `tailwind-4` |
| React 19 router/query | `react-19-tanstack` |

历史 unpkg、router、eager、ESM 等 demo 的关键产物验证已迁移到 Rslib/CDN package smoke。
这类能力通过构建产物、HTTP 静态服务、全局变量和声明文件检查覆盖，不再依赖多个旧 app 同时启动。

### 升级检查建议

升级后建议至少运行：

```bash
pnpm emp build
pnpm emp dev
```

在 EMP 仓库内回归 v4 示例和运行时能力时，使用：

```bash
corepack pnpm@10.33.0 apps:acceptance
corepack pnpm@10.33.0 test:library-output
```

其中 `apps:acceptance` 会覆盖默认 app 构建、真实 Module Federation 浏览器烟测和 library output 烟测。

## 从 EMP2 升级到 EMP3

## 背景
`EMP3`已经处于发布状态并落地在线上项目之中，经过多项目与`EMP2`比对，首次构建速度提升`80%`，二次构建速度提升`60%`，产物包体积缩小`25%`以上。

## 升级步骤

### 1.卸载 `@efox/emp`

<PackageManagerTabs command="remove @efox/emp" />

:::tip
项目存在的其他 `@efox/xxx` 依赖，都可以一并移除。
:::

### 2.安装 `@empjs/cli`

<PackageManagerTabs command="add @empjs/cli -D" />

### 3.修改 `package.json`
`EMP3` 建议使用 ESM 模块系统，在package.json中添加如下配置
```json title="package.json"
"type": "module",
```

### 4.修改 `emp-config.js`
:::tip
配置文件名也可以使用 typescript `emp.config.ts`文件
:::

#### 4.1 模块引入格式调整
`EMP2.x`的配置默认使用 `commonjs` 模块规范，升级 `EMP3` 首要是把`commonjs`的模块引用方式修改为`ESM`的引用方式，则 `require` 和 `module.exports` 或 `exports` 改为 `import` 与 `export` 或 `export default`

#### 4.2 模块返回结果调整
`EMP3`配置需要返回 `defineConfig` 函数的执行结果，调整如下:
```javascript title="emp-config.js
import { defineConfig } from '@empjs/cli'

export default defineConfig(async ({ mode }) => {
// 构建逻辑
    return {
        // 最终配置
    }
}
```

详细配置可以参考 [DEMO](https://github.com/empjs/emp/blob/v3/demos/react18/react-host/emp.config.ts)，其中用到的相关插件可以在 [插件](zh/plugin/index) 中查看使用说明。

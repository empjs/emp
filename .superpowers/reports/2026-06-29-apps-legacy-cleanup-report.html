删除策略

  • 未纳入验收矩阵的历史 demo：直接退役，避免 apps 面继续膨胀。
  • 根脚本绑定的旧链路：先移除入口脚本，再删除对应项目。

验收结论

  • apps 目录已经从 49 个真实项目收敛到 13 个，并与脚本识别结果一致。
  • Module Federation 不是只做构建检查，已通过 host + remote 的浏览器真实烟测。
  • Tailwind CSS 默认样例保留 v4，旧 v2/v3 demo 退役，文档同步。
  • React router、unpkg 类历史 demo 的关键输出改由 Rslib/CDN package smoke 覆盖。
  • Legacy lib 的声明文件产物已与 package metadata 对齐，避免干净构建后 types 缺失。
  • 根脚本和公开文档不再引用已删除 apps，后续新增旧链路会被 Rstest 规则拦截。

剩余注意项

  • 本轮是 apps cleanup 和验收面收敛，未改变 packages 的发布范围和独立版本线。
  • @empjs/plugin-tailwindcss2 / @empjs/plugin-tailwindcss3 仍按包维度保留，不再通过 app demo 作为默认样例。
  • 如果后续需要恢复某个历史 demo，应先把它加入 catalog、Rstest 规则和 acceptance 矩阵。
